Archive Bombs
eZIPlode!
History on Archive Bombs: 42.zip
eZIPlode is a script that will generate a zip file with an absurd amount of data inside.

I chose to stick with the very standard 32-bit vanilla zip, so some limitations apply:

	-Files can only be 4.3 GB in size

	-Can only have 65535 files

This means we are limited to a mere 256 TB :(

ZIPmouth

ZIPmouth allows you to add encrypted data into most zip files.


This encrypted data is encapsulated with a fake zip header


Even if someone were to extract the entire archive, the data would not be contained in any of the extracted files

Hiding in Illness (Virus Decoys)
HIVsneeze
HIVsneeze generates a user-defined amount of "virus's"

These are false-positives; they are just signatures (extracted from ClamAV's definitions)

Many file types generated will actually still function
	-exe's will execute

	-pdf's will open a pdf in your reader

	-zip files resemble r.zip

	-etc...
Current Features/Usage

	-User defined amount of files

	-Can define filename and extension

	-"spread" option will randomize file extensions when using
              many files

	-"signatures" option allows you to pack more than 1 signature
              in each file

	-Ability to add random padding data

	-The "Suckerpunch" option, a carefully crafted 'signature' that
              scans high across most vendors
Why HIVsneeze?

	-Make a target look hostile/damaged/infected

	-A more powerfull EICAR (test your AV)

	-Forensic Saturation when used with HIVasaur

HIVasaur
HIVasaur does everything HIVsneeze can, but it also injects encrypted steganography.

For obvious reasons, this script works with single files; instead of 'sneezing' a batch.

When using HIVasaur, I recommend also doing a big hivsneeze (with padding) to make forensics more nightmarish.

File Carving Mishaps
MagicBomb
First, let's talk about how Scalpel works... Then, how we an abuse it
Usage: -Magicbomb takes a scalpel.conf as an input to it's header/footer generation.

-It spits out all of the headers, then all of the footers.

-You tell the script how many times you would like this deviant behavior to repeat.

-If you'd like, you can insert random padding between the headers and footers.
MP3 mp3.conf:
	mp3	y	8000000	ID3


$ magicbomb.pl --multiplier=10 mp3.conf out.dd

MagicBomb --asour

-MagicBomb --asour is an extension of the same MagicBomb script.

-This option adds encrypted steganography between the headers and footers.

-The steganography has it's own header/footer (the defaults are configureable).
